# Live Product Inspection

Start the app through `den-serve` so it binds LAN-facing and records a managed
session:

```bash
den-serve up asha-rulebench -repo /home/dev/asha-rulebench
```

The interaction-first campaign uses human choices against real public content;
it does not retain a scripted product journey. Tester automation remains
focused on the product boundary gate. Artifact, provenance, and replay remain
secondary inspection surfaces rather than the primary play evidence.

Use the printed `local:` URL on this machine or report the printed `lan:` URL
for human inspection from another machine.

Record milestone screenshots when the claim is visual, visible outcomes, exact
dependency revisions, and explicit non-claims in the task handoff.

Completion evidence report:

```text
Live session:
Managed URL:
Content and dependency revisions:
Screenshots inspected:
Rendered behavior observed:
Supporting checks:
Non-claims / residual risk:
```
